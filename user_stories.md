# 📋 Gachi — 유저 스토리 (User Stories)

> **프로젝트명**: Gachi — 캠퍼스 익명 활동 매칭 서비스  
> **작성일**: 2026년 4월 30일

---

## 액터 정의

| 액터 | 설명 |
|---|---|
| **일반 사용자 (User)** | 대학생. Google 계정으로 로그인하여 모임을 만들거나 참여하는 사용자 |
| **관리자 (Admin)** | 서비스 운영자. Django Admin을 통해 전체 데이터를 관리하고 대시보드를 확인하는 관리자 |

---

## Release 1 — 인증 + 모임 기초

### US-01. Google OAuth 로그인

> **As a** 대학생 사용자,  
> **I want to** Google 계정으로 간편하게 로그인할 수 있기를 원한다.  
> **So that** 별도의 회원가입 없이 빠르게 서비스를 이용할 수 있다.

**인수 조건 (Acceptance Criteria)**
- [x] Google 로그인 버튼을 클릭하면 Google OAuth 인증 페이지로 이동한다.
- [x] 최초 로그인 시 닉네임이 자동으로 생성된다 (예: `user_abcdef`).
- [x] 로그인 완료 후 모임 목록 페이지(`/meetings/`)로 리다이렉트된다.
- [x] 이메일은 서버에만 저장되며, 다른 사용자에게 절대 노출되지 않는다.

---

### US-02. 재로그인 시 계정 유지

> **As a** 기존 사용자,  
> **I want to** 로그아웃 후 다시 로그인해도 기존 데이터(채팅, 모임, 프로필)가 유지되기를 원한다.  
> **So that** 매번 새로운 계정이 생성되는 불편함 없이 서비스를 계속 이용할 수 있다.

**인수 조건**
- [x] 같은 Google 이메일로 재로그인하면 기존 계정에 자동으로 연결된다.
- [x] 이전에 참여했던 모임, 채팅 기록, 매너온도가 그대로 유지된다.

---

### US-03. 모임 생성

> **As a** 사용자,  
> **I want to** 카테고리, 제목, 내용을 입력하여 새로운 모임을 개설할 수 있기를 원한다.  
> **So that** 같은 취미를 가진 캠퍼스 친구들이 모임의 상세 내용을 보고 참가할 수 있다.

**인수 조건**
- [x] 운동, 게임, 스터디, 식사 중 하나의 카테고리를 선택할 수 있다.
- [x] 모임의 제목과 상세 내용을 텍스트로 입력할 수 있다.
- [x] 모임 생성 시 호스트가 자동으로 첫 번째 멤버로 등록된다.
- [x] 생성 완료 후 모임 목록 페이지로 이동한다.

---

### US-04. 모임 목록 조회 및 필터링

> **As a** 사용자,  
> **I want to** 생성된 모임 목록을 확인하고 제목과 내용을 볼 수 있기를 원한다.  
> **So that** 내 관심사에 맞는 모임을 상세 내용을 통해 빠르게 찾을 수 있다.

**인수 조건**
- [x] 전체 모임 목록이 카드 형태로 표시된다.
- [x] 각 카드에 모임 제목, 모임 내용, 호스트 닉네임, 장소, 일정, 참여 인원이 표시된다.
- [x] 전체/운동/게임/스터디/식사 탭으로 카테고리 필터링이 가능하다.
- [x] 최신순으로 정렬된다.

---

## Release 2 — 매칭 + 소통

### US-05. 지도에서 모임 장소 선택

> **As a** 모임 호스트,  
> **I want to** 카카오맵에서 지도를 클릭하여 모임 장소를 지정할 수 있기를 원한다.  
> **So that** 참여자들이 정확한 위치를 확인할 수 있다.

**인수 조건**
- [x] 모임 생성 화면에 카카오맵이 표시된다.
- [x] 지도를 클릭하면 마커가 표시되고, 해당 위치의 주소가 자동으로 표시된다.
- [x] 선택된 좌표(lat, lng)와 주소가 DB에 저장된다.

---

### US-06. 모임 일정 선택

> **As a** 모임 호스트,  
> **I want to** 달력과 시간 선택기를 통해 모임 일정을 지정할 수 있기를 원한다.  
> **So that** 참여자들이 모임 시간을 정확히 알 수 있다.

**인수 조건**
- [x] 날짜와 시간을 함께 선택할 수 있는 타임피커가 제공된다.
- [x] 과거 날짜는 선택할 수 없다.
- [x] 한국어 로케일이 적용된다.

---

### US-07. 모임 참가

> **As a** 사용자,  
> **I want to** 다른 사람이 만든 모임에 참여할 수 있기를 원한다.  
> **So that** 관심 있는 활동에 함께할 수 있다.

**인수 조건**
- [x] 모임 목록에서 "참여하기" 버튼을 클릭하면 해당 모임의 멤버로 등록된다.
- [x] 이미 멤버인 경우 바로 채팅방으로 이동한다.
- [x] 정원(기본 4명)이 가득 찬 모임에는 참여할 수 없으며, 에러 메시지가 표시된다.
- [x] 참가 후 자동으로 채팅방으로 이동한다.

---

### US-08. 인앱 채팅

> **As a** 모임 멤버,  
> **I want to** 모임 멤버들과 실시간으로 채팅할 수 있기를 원한다.  
> **So that** 모임 전에 일정과 세부사항을 조율할 수 있다.

**인수 조건**
- [x] 채팅방에 입장하면 이전 메시지 기록이 표시된다.
- [x] 메시지를 전송하면 다른 멤버에게 실시간으로 표시된다 (Long Polling).
- [x] 내 메시지와 상대 메시지가 시각적으로 구분된다 (좌/우 정렬, 색상 차이).
- [x] 보낸 사람의 닉네임과 전송 시간이 표시된다.
- [x] 모임의 멤버가 아닌 사용자는 채팅방에 입장할 수 없다.

---

## Release 3 — 신뢰 + 관리

### US-09. 매너온도 평가

> **As a** 모임 참여자,  
> **I want to** 모임이 끝난 후 다른 참여자를 평가할 수 있기를 원한다.  
> **So that** 좋은 매너의 사용자를 알아볼 수 있고, 건강한 커뮤니티가 유지된다.

**인수 조건**
- [x] 채팅방에서 각 참여자 옆에 👍(칭찬) / 👎(아쉬워요) 버튼이 표시된다.
- [x] 👍 클릭 시 상대의 매너온도가 +0.5°C 상승한다.
- [x] 👎 클릭 시 상대의 매너온도가 -0.5°C 하락한다.
- [x] 같은 모임에서 같은 사람을 2번 이상 평가할 수 없다 (조작 방지).
- [x] 자기 자신은 평가할 수 없다.
- [x] 평가 완료 시 현재 매너온도가 알림으로 표시된다.

---

### US-10. 유저 신고

> **As a** 사용자,  
> **I want to** 부적절한 행동을 하는 유저를 신고할 수 있기를 원한다.  
> **So that** 안전한 커뮤니티 환경이 유지된다.

**인수 조건**
- [x] 채팅방 또는 유저 상세 페이지에서 🚨 신고 버튼을 클릭할 수 있다.
- [x] 신고 사유를 선택할 수 있다 (욕설/비하, 노쇼, 부적절한 만남 목적, 기타).
- [x] 자기 자신은 신고할 수 없다.
- [x] 신고 완료 시 확인 메시지가 표시되고 모임 목록으로 이동한다.

---

### US-11. 신고 누적 자동 밴

> **As a** 관리자,  
> **I want to** 신고가 5회 이상 누적된 유저가 자동으로 차단되기를 원한다.  
> **So that** 수동 관리 없이도 악성 유저가 자동으로 제재된다.

**인수 조건**
- [x] 동일 유저에 대한 신고가 5회 누적되면 해당 계정이 자동으로 비활성화된다.
- [x] 비활성화된 유저는 로그인할 수 없다.
- [x] 신고 처리는 서버 시그널(Signal)을 통해 자동으로 이루어진다.

---

### US-12. 내 프로필 조회

> **As a** 사용자,  
> **I want to** 내 프로필에서 매칭 이력과 즐겨찾기한 멤버를 확인할 수 있기를 원한다.  
> **So that** 과거 모임 기록을 돌아보고 좋았던 파트너를 다시 찾을 수 있다.

**인수 조건**
- [x] 상단 네비게이션에서 "내 프로필" 링크를 통해 접근할 수 있다.
- [x] 내 닉네임과 현재 매너온도가 표시된다.
- [x] 참여했던 모임 목록이 카테고리, 제목, 장소, 날짜와 함께 표시된다.
- [x] 즐겨찾기한 멤버 목록이 닉네임, 매너온도와 함께 표시된다.

---

### US-13. 유저 상세 프로필 및 즐겨찾기

> **As a** 사용자,  
> **I want to** 다른 유저의 프로필을 확인하고 즐겨찾기에 추가/해제할 수 있기를 원한다.  
> **So that** 좋았던 모임 파트너를 기억하고 나중에 다시 찾을 수 있다.

**인수 조건**
- [x] 채팅방에서 닉네임을 클릭하면 해당 유저의 상세 프로필 페이지로 이동한다.
- [x] 상대의 닉네임과 매너온도가 표시된다.
- [x] ⭐ 즐겨찾기 추가/해제 버튼이 제공된다.
- [x] 🚨 신고하기 버튼이 제공된다.
- [x] 자기 자신의 프로필에서는 즐겨찾기/신고 버튼이 표시되지 않는다.

---

### US-14. 관리자 밀집도 대시보드

> **As a** 관리자,  
> **I want to** 모든 모임의 위치 분포를 지도에서 한눈에 확인할 수 있기를 원한다.  
> **So that** 어느 지역에서 매칭이 활발한지 분석하고 서비스 운영 전략을 수립할 수 있다.

**인수 조건**
- [x] Django Admin 내에서 밀집도 대시보드에 접근할 수 있다.
- [x] 카카오맵에 모든 모임 위치가 마커로 표시된다.
- [x] 마커 클러스터링이 적용되어 밀집 지역이 시각적으로 구분된다.
- [x] 마커에 마우스를 올리면 모임 정보(카테고리, 장소명)가 표시된다.
